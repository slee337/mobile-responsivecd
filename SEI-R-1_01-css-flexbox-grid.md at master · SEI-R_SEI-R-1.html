Setup

  • For this lesson, we'll be using VS Code and the Live Server extension to learn about Flexbox & CSS Grid.

  • Please create a new HTML and CSS file on your local machine in your code folder and open them in VS Code:

    mkdir css-flexbox-grid && cd css-flexbox-grid
mkdir css
touch index.html css/style.css
code .

  • Use the ! emmet abbreviation in index.html to create HTML boilerplate.

  • Link up your CSS file by adding this to the index.html's head:

    <link rel="stylesheet" href="css/style.css">

  • Finally, add a bit of starting CSS inside of style.css:

     html {
   box-sizing: border-box
 }

 /* The Universal Selector */
 *, /* All elements*/
 *::before, /* All ::before pseudo-elements */
 *::after { /* All ::after pseudo-elements */
   /* height & width will now include border & padding by default
      but can be over-ridden as needed */
   box-sizing: inherit;
 }
 
 body {
   background-color: gray;
   font-family: sans-serif;
   margin: 0;
 }

Intro to Flexbox & CSS Grid

  • As a front-end developer, you will be required to layout elements on web pages precisely.

  • Before Flexbox & CSS Grid, building a web page layout was cumbersome and overly complicated - Flexbox & Grid were game-changers.

  • The capabilities of Flexbox & Grid complement each other. Using both side-by-side and even nesting one within the other is common.

Intro to Flexbox & CSS Grid

  • The difference between Flexbox and CSS Grid is how they lay out their children:

Flexbox

Why Use Flexbox?


Flexbox excels at assisting devs with the following tasks:

  • Vertically centering content & elements within a container element

  • Spacing child elements within a container uniformly

  • Making the height of child elements laid out in columns the same even though they have a different amount of content

  • Creating responsive designs without using media queries

Flexbox Fundamentals


  • We use a CSS display: flex; declaration to make an element a flex container, for example:

     section {
   display: flex;
 }

    The above would make all <section> elements flex containers, and all direct children become flex items.

  • Let's open in a separate tab and briefly review what has become the de facto guide to Flexbox.

Your First Flexbox


  • We're going to make a navigation bar using Flexbox.

  • Add the following markup for the navbar inside of the <body>:

     <nav>
   <div>Home</div>
   <div>About</div>
   <div>Widgets</div>
   <div>Log out</div>
 </nav>

Your First Flexbox


  • Run the HTML file in a live server to check it out - definitely not what we want to see!

  • Use your dev tools to verify that the <nav> & <div> elements are block elements that take up all available width.

  • Which element do we need to make the flex container?

Your First Flexbox


  • Let's make the <nav> a Flexbox:

     nav {
   display: flex;
 }

  • Run again, and we can make the following observations:

    • The flex items are laid out horizontally in a row - this is the default layout of a flex container.
    • The <div> elements have become flex items. They are no longer block elements - their width has collapsed to that of their content, and they are willing to sit side-by-side with other elements.

Your First Flexbox


Let's make the Flexbox better!

  • The navigation bar must:
    • Be 60px in height
    • Have a background color of #a2cbb6.
  • The menu items in the navbar need:
    • A font size of 20px
    • A bold font-weight
    • A margin of 10px to the left and right
    • A text color of #f3dba8

Your First Flexbox


  • A flex container has a flex-direction property that defines the direction of its main axis.

  • There are four values:

    • row - the default
    • row-reverse
    • column
    • column-reverse

  • Let's check them out by adding a flex-direction to the <nav>.

Your First Flexbox


  • A flex container also has a cross axis, which represents the opposite direction of its main axis.

  • For example, if the flex-direction is row (the default):

    • the main axis is horizontal
    • the cross axis is vertical

  • If the flex-direction is column, they flip:

    • the main axis is vertical
    • the cross axis is horizontal

Your First Flexbox


  • These concepts come into play when it comes to sizing and layout properties, such as:

  • justify-content: Controls alignment for the main axis

  • align-items: Controls alignment for the cross axis

  • Let's refer to the Guide to Flexbox we opened to see the options we have and experiment a bit!

Your First Flexbox


  • With the following alignment properties set:

     nav {
   align-items: center; /* changed from the default of stretch */
   background-color: #a2cbb6;
   display: flex;
   flex-direction: row; /* default */
   height: 60px;
   justify-content: flex-start; /* default */
 }

    The navbar is not looking too bad (minus the color, we'll fix that soon)

Your First Flexbox


  • Let's say you want the Log out menu item to be right-aligned:

  • You could wrap the others with another element and set justify-content to space-between.

  • Or, we can use this bit of CSS goodness:

     nav > div:last-child {
   margin-left: auto;
 }

Review Questions - Flexbox


  • When an element has a CSS property of display: flex;, that element becomes a flex __________.

  • When an element has a CSS property of display: flex;, its direct children become flex __________.

  • What value is the default for the flex-direction property?

  • Is it justify-content or align-items that controls the alignment along the cross axis?

Your First Flexbox ... A Brief Detour


So our navbar is spaced right, but the colors look terrible. Find out why.

  • Large text (24px or larger) and UI elements should have a contrast ratio of at least 3:1

  • Text smaller than 24px should have a contrast ratio of at least 4.5:1

  • Ideally, large text should have at least a 4.5:1 contrast ratio, and small text should have at least a 7:1 contrast ratio.

Your First Flexbox ... A Brief Detour


Let's make this navbar accessible!

  • Pick a new accessible navbar background color and text color.

  • No eye for color? There are so many resources out there to help you out! These are a couple of cool ones: ColorSpace and Coolors.

Take a few minutes to find a new color combo before we go back to flexboxing. Make sure your color combination is accessible - verify it here

Can't find a good color combo? The rest of this guide uses #21313F as the background and #64F8EF as the font color in the nav.

Your First Flexbox ... A Brief Detour

  • Our navbar is looking good, but for our mock-up to be complete, we should add a placeholder for a profile picture. It will appear after the logout button.

  • We don't have any photos handy though, so we're going to use Picsum Photos, which bills itself as "the Lorem Ipsum for photos" to generate a placeholder.

  • Add this line to your index.html file as the new last child of the nav element:

    <img src="https://picsum.photos/40/" alt="A randomly generated placeholder">

  • This line will fetch a random 40px square image from their collection - visit their site for the full documentation.

Your First Flexbox ... A Brief Detour

This broke a few things on our navbar, unfortunately. Let's fix them before we move on.

  • All the flex items are now on the left side of the navbar.

    • This is happening because in nav > div:last-child, the :last-child pseudo-class is no longer selecting a div, but an img causing the rule to be invalid.

    • Fix it by modifying the pseudo-class to be :last-of-type instead. The div containing the Log out text and the placeholder image will now move to the right side of the navbar.

    • Changing the selector to nav > *:nth-last-child(2) would have had the same result.

  • Now our image is pushed up to the right side of the browser window. Give it a margin of 10px on the left and right and a border-radius of 7px to round off the corners while we're here:

    nav > img {
  border-radius: 7px;
  margin: 0 10px;
}

CSS Grid

Why use CSS Grid?


CSS Grid is a great option when you have:

  • A page layout like this (or as complex as you'd like):

  • Any other "components" that benefit from a grid-type arrangement such as a profile card. In other words, CSS Grid doesn't have to apply to the whole page - it can be useful for laying out smaller "components" as well.

  • A need to have guaranteed white space on a page.

CSS Grid Fundamentals


  • Unlike Flexbox, CSS Grid lays out its grid items in two-dimensions.

  • CSS Grids have the concept of the following:

    • Tracks
    • Cells
    • Areas
    • Gaps

  • Let's examine a diagram to visualize these components:

CSS Grid Fundamentals


  • As you might expect, there are plenty of CSS Grid-related properties and values.

  • Here's the CSS Grid equivalent of that Flexbox guide we used earlier.

  • Let's open it up in a new tab and take a peek.

  • Here is the outcome of this exercise:

Your First CSS Grid


  • To try out CSS Grid, we'll continue to work in the existing file.

  • The following CSS turns the <body> element into a grid container:

     body {
   display: grid;
   height: 100vh;
   margin: 0;
   font-family: Helvetica;
 }

  • Using height: 100vh; will make the <body> fill the height of the browser window so that the <footer> is at the bottom.

Your First CSS Grid


  • Let's add the additional HTML required by the UI:

     <body>
   <nav>
     <div>Home</div>
     <div>About</div>
     <div>Widgets</div>
     <div>Log out</div>
   </nav>
   <aside>Side bar</aside>
   <main>Main Content</main>
   <footer>Footer</footer>
 </body>

Your First CSS Grid


Now for some style

  • Let's change the color of the elements we just added so that we can more easily see them:

     aside {
   background-color: #38b18a;
 }
 
 main {
   background-color: #92d97c;
 }
 
 footer {
   background-color: #f9f871;
 }

Your First CSS Grid


  • One more stylistic touch. What if we want to center the text in those elements both horizontally and vertically?

  • Wanting to center/center content is so common, let's create a class that will make any element with that class a Flexbox:

     .grid-cntr {
   display: grid;
   place-items: center;
 }

  • With the class defined - go ahead and add it to the <aside>, <main> & <footer> elements.

Your First CSS Grid


  • Using Chrome DevTools to explore the page's elements, we can make the following observations:

    • A CSS Grid has a single column by default.
    • Each grid item (direct child) was placed alone on a row by default.

  • Now let's define the columns and rows necessary to layout our page as desired. Go back, look at the UI we want to design and answer these questions:

    • How many columns will we need to define?
    • How many rows?

Your First CSS Grid


  • Okay, let's define these columns and rows by adding these lines to our body rule:

     grid-template-columns: 1fr 4fr;
 grid-template-rows: auto 1fr auto;

  • The fr unit is used by CSS Grid to represent a fraction of the available space. So in our layout, the first column will be 1/5th the width of the window.

  • The auto unit will let the content in that row determine the size of that row.

Your First CSS Grid


  • The live server shows that we've made a mess. Notice how each grid item gets placed in each cell across the columns from left to right - the default behavior.

  • Next, let's make both the <nav> and the <footer> span two columns each.

Your First CSS Grid


  • There are a couple of ways to make grid items cover rectangular grid areas.

  • One way is to define grid-template-areas on the grid container; then using the grid-area property on the grid item.

  • For now, we'll look at another option.

Your First CSS Grid


  • The grid-column CSS property determines which grid lines a grid item starts and ends on.

  • For example:

     nav, 
 footer {
   grid-column: 1 / 3;
 }

  • Grid-lines start counting up from 1 (not zero).

Your First CSS Grid


  • We can also use span to specify how many columns we want to span:

     nav, footer {
   grid-column: span 2;
 }

  • Unsurprisingly, there's a grid-row property as well.

  • Both grid-column & grid-row are shorthand for grid-column-start & grid-column-end, and grid-row-start & grid-row-end, respectively.

Your First CSS Grid


Now that the footer element is on a row by itself, we can give it a height.

  • Add this line to the footer rule: height: 40px;

  • The auto value we gave to this row when we wrote our grid-template-row declaration means that this row will now expand to be 40px tall.

  • The total height of our body is still 100vh - so the middle row, which has a value of 1fr, will shrink to accommodate the footer's new size.

Your First CSS Grid


Let's break the second column into two sections to demonstrate some of the issues and benefits you may find when using grid.

  • In the body rule change this declaration: grid-template-columns: 1fr 4fr; to this: grid-template-columns: 1fr 2fr 2fr;. This change will add another column.

  • You do: Notice that the nav and the footer elements no longer reach the end of their rows - fix this issue.

  • You do: Add this declaration: grid-column: 3/4; to the main rule.

Your First CSS Grid


We just made white space (well, gray space) on our page!

  • The area in the second row between lines 2 and 3 will remain white space; no content can enter that area.

  • You do: Expand the main content to fill the white space.

Your First CSS Grid


Future browsers will support a value subgrid on the grid-template-columns and grid-template-rows properties that will enable this behavior.

  • The World Wide Web Consortium continually develops CSS features for browser creators to implement! Can I Use is a great resource to ensure the CSS you write is compatible with browsers.

Your First CSS Grid


The grid-template-areas property allows us to name grid areas and then place elements inside those areas. This property keeps us from having to specify column/row line numbers for our elements, and use names instead.

  • Add these template areas to the body rule:

    grid-template-areas: 
"nav nav nav"
"aside main main"
"footer footer footer";

  • Notice that adding this doesn't break our existing layout. These can be used in conjunction with placing items by grid line number.

  • You do: Add a grid-area property to the nav, aside, main, and footer rules.

    • Also, add a corresponding value for each. Do not surround the value in quotes.

    • Remove the grid-column properties and corresponding values.

Your First CSS Grid


  • The last thing we'll look at in CSS Grid is the gap property, which specifies the size of the grid's lines.

  • Update the CSS of the <body> (grid container) to the following:

     body {
   display: grid;
   grid-template-columns: 1fr 2fr 2fr;
   grid-template-rows: auto 1fr auto;
   gap: 5px;  /* specifies width of the grid's lines */
   ...

  • Note that the gaps cannot be styled - the background of the parent element shows through instead.

  • The gap property is also supported when used with Flexbox in some newer browsers! Check out Can I Use.
